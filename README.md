# Close Controller

Close Controller is a local-first web app for recurring accounting operations across multiple clients. It is designed for month-end close, quarter-end close, year-end close, audit PBC tracking, monthly tax work, and one-off accounting projects.

## Stack

- Next.js 16 with TypeScript
- React
- Tailwind CSS 4
- Prisma ORM
- SQLite for local development
- zod validation

## What is included

- Client management with archive support
- Internal team accounts with sign-in
- Client-to-team memberships for collaboration
- Reusable workflow templates
- Template task setup with due date rules, dependencies, carryforward behavior, evidence flags, reviewer flags, and default teammates
- Period generation from templates
- Rollforward to the next period with optional carryforwards
- Task management for status, teammate assignment, priority, notes, blockers, reviewer signoff, and evidence links
- Audit PBC tracking by client and period
- CSV/XLSX import preview, column mapping, validation, and commit flow
- Demo seed data
- Sample import files in `sample-data/`

## Local setup

1. Install dependencies:

```bash
npm install
```

2. Copy `.env.example` to `.env` if needed and ensure `DATABASE_URL` points to the local SQLite file:

```bash
DATABASE_URL="file:./dev.db"
```

3. Generate the Prisma client:

```bash
npm run db:generate
```

4. Apply the SQLite migration history:

```bash
npm run db:deploy
```

5. Seed demo data:

```bash
npm run db:seed
```

6. Generate sample import files:

```bash
npm run sample-data
```

7. Start the app:

```bash
npm run dev
```

## Import flow

1. Go to `/imports`
2. Upload a CSV or XLSX file
3. Choose import type:
   `TASK_BATCH`, `TEMPLATE`, or `PBC_ITEMS`
4. Review the preview page
5. Adjust flexible column mapping or apply a saved mapping preset
6. Optionally save the current mapping for reuse on future files with the same style
7. Review the preview grid, which shows original row values, mapped values, validation status, and row-level errors
8. Resolve validation errors
9. Commit the batch into:
   a reusable template, one-time imported tasks in a period, or audit PBC items

Supported mapping examples include:

- `Task`, `Item`, `Description` -> task title
- `Assigned To`, `Owner` -> owner
- `Due`, `Due Date` -> due date
- `PBC #`, `Request #` -> request number

## Rollforward behavior

- Rollforward starts from an existing period instance
- The user selects the target period label and explicit target dates
- Active template tasks are regenerated from the same template and marked as normal generated tasks
- Prior period history is not mutated
- Carryforward options support incomplete tasks, blocked items, note copying, and completed one-time exclusion
- Carryforward items reference the original prior-period task instance
- Recurring template tasks are not duplicated as extra carryforward items when the target period already generated that template task

## Main routes

- `/` dashboard
- `/clients` clients and assignments
- `/team` internal users and collaboration setup
- `/templates` template library
- `/periods` period generation and register
- `/periods/[id]` task detail and rollforward
- `/imports` file upload and import history
- `/imports/[id]` mapping, validation, and commit
